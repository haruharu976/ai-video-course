よくあるエラー一覧
command not found: node CH.02 STEP2▶
zsh: command not found: node

原因：Node.jsがインストールされていない。
解決：macOSなら brew install node を実行。Homebrewが入っていない場合は nodejs.org からLTS版をダウンロード。
Windowsは nodejs.org から .msi インストーラーをダウンロード。

Error creating git repository / empty ident name CH.02 STEP3▶
Error: Could not create git repository
fatal: empty ident name (for <user@host>) not allowed

原因：gitにユーザー情報が設定されていない。
解決：以下を実行してから再度 npx create-video@latest を実行する。
git config --global user.email "you@example.com"
git config --global user.name "Your Name"

npm error 404 Not Found — create-remotion CH.02 STEP3▶
npm error 404 Not Found - GET https://registry.npmjs.org/create-remotion

原因：コマンド名が間違っている。
解決：create-remotion ではなく create-video@latest が正しい。
正しいコマンド → npx create-video@latest my-video

EACCES: permission denied CH.02 STEP3▶
npm error code EACCES
npm error syscall mkdir
npm error path /usr/local/lib/node_modules

原因：権限がないディレクトリに書き込もうとしている。
解決（macOS）：コマンドの先頭に sudo をつけて実行。
または nvm（Node Version Manager）を使うと権限問題が起きにくい。

ポート3000が使用中 / address already in use CH.02 STEP4▶
Error: listen EADDRINUSE: address already in use :::3000

原因：別のプロセスがポート3000を使っている。
解決：npx remotion studio --port 3001 で別ポートで起動する。
またはポート3000を使っているプロセスを終了する → lsof -ti:3000 | xargs kill -9

Cannot find module 'remotion' CH.02 STEP4▶
Error: Cannot find module 'remotion'
Require stack: - /path/to/my-video/src/...

原因：node_modulesがインストールされていない、またはプロジェクトフォルダの外で実行している。
解決：cd my-video でフォルダに入ってから npm install を実行。その後 npx remotion studio を再実行。

ModuleNotFoundError: No module named 'moviepy' CH.04▶
ModuleNotFoundError: No module named 'moviepy'

原因：MoviePyライブラリがインストールされていない。
解決：pip install moviepy または pip3 install moviepy を実行。
仮想環境を使っている場合はアクティベートを確認 → source venv/bin/activate

AIへの質問を上手くするコツ
01
エラー全文を貼る

「エラーが出ました」だけではAIは解決できない。ターミナルのエラーメッセージを全部コピーして貼り付けよう。

02
環境情報を伝える

OS（Mac/Windows）とNode.jsのバージョンを必ず書く。同じエラーでもOSによって解決策が違う。

03
何をしたかったか書く

「○○をしようとした」というゴールを伝えると、AIが正しい方向の解決策を出してくれる。

04
3つのAIを試す

1つのAIで解決しなければ他のAIに同じテンプレを送ってみよう。得意分野が違うので突破口が見つかる。

05
解決策をそのまま実行

AIが提示したコマンドはまずそのままコピーして実行してみる。「なぜか」は後で聞けばOK。

06
新しいエラーも貼る

解決策を試して新しいエラーが出たらそれもAIに貼る。チャット履歴を続けると文脈を引き継いでくれる。
